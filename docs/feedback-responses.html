# Comment Response
1 While each section of the response for each activity is concise, there is still a lot of information overall, which can be rather timely to read and understand. There may need to be a rethink of the layout of content, as well as content used. Implemented
The response layout has been redesigned with a clearer card format. The agent is now capped at a maximum of 5 activities per response, each kept to a few short bullet points rather than a full transcript of the notes. Admin controls for response length (Brief / Standard / Detailed) are also in place.
2 For this particular prompt, the word 'activities' to a teacher means something hands-on, fun and engaging. Implemented
The system prompt now explicitly defines an activity as hands-on, pupil-facing and fun — distinct from a teaching strategy. The agent applies this definition whenever "activities" appears in a prompt.
3 Outputted activities 1, 2 and 7 are what I would expect from this prompt, being that they have some sort of fun gimmick such as a spinning wheel. Confirmed
This validates the activity definition. The system prompt has been aligned to this standard so only outputs of this type are returned when activities are requested.
4 Outputted activities 3, 4 and 8 are what we would consider teaching strategies, as they are methods that we would expect students to consistently use. Implemented
The agent is now instructed to separate activities from teaching strategies and label them accordingly. Teaching strategies should only appear if specifically requested or clearly labelled as such.
5 Outputted activity 5 makes little sense in just the written form, I had to find the activity in the teacher notes and see the picture for it to be understandable. Implemented
The knowledge base stores text only — images inside slides cannot be displayed. Two things now address this: (1) the agent flags visual-dependent activities with a note to open the original document; (2) every section carries an inline clickable link to the exact source document (a secure, time-limited URL), so the teacher opens the original slide in one click, right where it is relevant.
6 Outputted activity 6 would also be considered a teaching strategy, however it did contain errors which are present in the teaching notes itself. Data Issue
The AI reproduces source material as written. Where errors exist in the teacher notes they will appear in the output. The fix requires correcting the source document, re-uploading to S3, and resyncing the knowledge base. This is a data quality task outside the AI system.
7 Outputted activity 3 was also found to be explained very differently from what is in the teacher notes, this in turn changed it from an activity in the teacher notes to just a teaching strategy in the outputted response. Implemented
Two changes address this: (1) the knowledge base now retrieves 12 chunks per query instead of 5, giving the agent more of the original text; (2) the system prompt instructs the agent to name and describe activities as they appear in the notes, not reinterpret them.
8 When a response is on a particular topic such as 'Number Bonds', the AI should also look at content from related lessons — Number Bonds to 5; 6 and 7; 8 and 9 — as the activities are interchangeable with minor numerical adjustments. Implemented
The system prompt now instructs the agent to search for related topic variants when a specific number set is mentioned, make multiple knowledge base queries if needed, and note when an activity comes from a related lesson.
9 I would also expect the outputted response to come up with new or variations of activities in the teacher notes, rather than just plucking information from the teacher notes itself. Implemented
The system prompt now explicitly instructs the agent to use retrieved materials as inspiration rather than text to reproduce, and to suggest practical variations or extensions beyond what is written in the notes.
10 Output requirements may vary from input to input (e.g. when an input asks for activities, it should produce responses based on the content in the THINK, SOLVE and DEVELOP slides). Feasible — To Do
The THINK / SOLVE / DEVELOP slide structure needs to be explained to the agent in the system prompt so it knows what to look for when generating activities. This is a prompt change that can be made once the slide structure is confirmed with the team.
11 Some responses, especially for activities and methods, may benefit from visual examples to help speed up the understanding of an activity. Some people, no matter how much they read something, they can never visualise it. Workaround In Place
The knowledge base is text-only and cannot render images from the teaching slides — generating original diagrams would require a dedicated image layer, out of scope for this phase. However, each section of the response now includes an inline clickable link to the original document, so the teacher reaches the relevant visual in one click (see comment 5).
